# 排程操作指引與任務拆分

## 排程操作指引

1. **建立排程** -- 解析意圖後直接呼叫 `create`，不需二次確認。**必須帶 `--thread-ts`**，讓排程結果回覆到原始 thread
2. **相對時間一律用 `--delay`**（如 `--delay 2m46s`），不要自己算 ISO datetime，避免時區錯誤
3. **修改排程** -- 呼叫 `update`，只傳需要修改的欄位
4. **取消排程** -- 呼叫 `delete`
5. **查看排程** -- 呼叫 `list` 或 `get`
6. **cron 表達式** -- 分 時 日 月 週（5 欄），時區 Asia/Taipei
7. **所有工具輸出 JSON**，方便解析和回報給使用者
8. **任務鏈** -- 先用 `create` 建立第一步（帶 trigger），再用 `chain-create --upstream <id>` 逐步串接
9. **鏈式任務失敗重試** -- 用 `create --delay 5s` 建新任務帶同樣 prompt，再 `chain-create` 接上剩餘步驟

## 任務拆分指引

當排程任務預估超過 25 分鐘（留 5 分鐘緩衝），應主動拆分為多步驟鏈式任務。

### 何時拆

- 涉及大量網路搜尋（如「蒐集 + 分析 + 報告」三段式任務）
- 需要處理多個獨立資料來源再彙整
- 使用者明確要求「分步驟」或「拆開做」

### 何時不拆

- 單純提醒（reminder）
- 預估 25 分鐘內能完成的任務
- 使用者明確要求一次完成

### 怎麼拆

1. 按「蒐集 -> 分析 -> 產出」的邏輯切步驟，每步控制在 20 分鐘內
2. 每步 prompt 指定中間結果路徑：`/tmp/slack-bot/chains/{描述性名稱}/step{N}.md`
3. 下一步 prompt 指定「先讀取上一步結果」再執行
4. 第一步用 `create`（帶 trigger），後續用 `chain-create --upstream`
