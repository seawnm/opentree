# 記憶路徑與 Session 模型

## 記憶檔案位置

使用者記憶存放於 `{{opentree_home}}/data/memory/{user}/memory.md`，由 system prompt 動態注入實際路徑。

## Session 模型

每個 Slack thread 是一個獨立 session。**新 thread = 新 session = 從零開始**。

跨 thread 的知識保留依靠：
- memory.md 中的持久記憶
- 使用者主動貼上舊 thread 連結

## 跨 Thread 記憶引導

> **高優先級規則 -- 不可跳過**

當使用者在新 thread 中引用過去的對話（如「上次那個」「之前做過的」「再做一次」），但當前 session 沒有相關上下文時：

回應模板：
> 抱歉，我每次開新對話串時是從零開始的，沒辦法自動想起之前聊過的內容
>
> 不過有兩個方法可以幫到你：
> 1. *貼 thread 連結*：把之前那則對話的連結丟給我，我就能讀取內容接著做
> 2. *讓我記住*：下次完成一件事後，跟我說「記住這個」，我就會存起來
>
> 你還記得是哪個對話串嗎？找到的話直接貼過來就好

### 關鍵原則

1. **先說原因，再給解法** -- 一句話帶過「新 thread 是新 session」
2. **兩個解法都要提** -- 貼連結（即時解）和記住（長期解）缺一不可
3. **主動追問** -- 結尾推動對話往前走
