# 記憶管理 SOP

## 記住流程

當使用者明確要求記住（「記住」「請記住」「記下」「remember」「以後記得」）：

1. **隱私檢查** -- 以下內容不可記錄，應婉拒並說明原因：
   - 密碼、password、passwd、pwd
   - API key、token（格式如 sk-、xoxb-、xoxc-、AKIA-）
   - 健康、宗教、政治等敏感推論
   - 其他使用者的個人資訊
2. **長度檢查**：若內容超過 3 句話或約 100 字，主動詢問是否改用摘要形式（單條建議 <=50 字）
3. Read memory.md（不存在則建立含四區段的空模板）；直接使用 Write 工具寫入，Write 工具會自動建立不存在的中間目錄
4. 在 `## Pinned` 區段末尾追加：`- 記憶內容 (YYYY-MM-DD)`
5. Write 完整 memory.md
6. 回覆確認已記住

## 忘記流程

當使用者要求忘記（「忘掉」「不要記」「刪除記憶」「取消記住」「forget」）：

1. Read memory.md
2. 在 `## Pinned` 區段找到包含對應關鍵字的行
3. 刪除該行
4. Write 完整 memory.md
5. 回覆確認已刪除

## 記憶展現指引

在互動中展現對使用者的了解：

1. **自然融入**（為主）：根據記憶調整回覆風格和內容，不需特別聲明
2. **適度提示**（偶爾）：話題相關時帶一句「你之前提到...」
3. **主動回顧**（累積觸發）：當記憶中近 7 天新增超過 5 條，摘要你的新理解讓使用者確認

## 優先級

衝突規則：Pinned > Core[explicit] > Core[inferred] > Episodes > Active

不要逐條列出記憶，除非使用者主動詢問。
