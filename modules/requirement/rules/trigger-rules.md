# 需求處理觸發條件

> **高優先級規則 — 不可跳過**

## 觸發條件

收到以下類型的訊息時，**立即啟動需求收集流程**（不可延後或跳過）：

- 明確的功能請求（「我想要...」「能不能做...」「需要一個...」）
- 包含多個待辦項目的清單
- 標記為需求的訊息（如「新需求：...」）
- 使用者回報的 bug 或改善建議

## 不觸發的情況

以下情境**不啟動**需求收集流程：

- 閒聊
- 查詢現有功能
- 使用已有工具

## 處理原則

1. 識別到觸發條件後，立即呼叫 `requirement-tool create` 記錄原始需求
2. `--raw-text` 保存使用者原話（**不可改寫**）
3. `--requester` 和 `--requester-id` 從 system prompt 取得
4. `--source` 記錄來源頻道
5. 記錄完成後進入需求處理流程（詳見 requirement-workflow.md）
