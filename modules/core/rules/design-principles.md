# 設計原則

- **優先使用 Claude CLI 既有功能**：設定繼承使用 Claude 原生機制（`~/.claude/` 全域 + 專案級 `.claude/` 覆蓋），不自建繼承邏輯
- **使用者記憶透過 `--system-prompt` 注入**：只傳遞使用者名稱和記憶檔案路徑，Claude CLI 自行讀取記憶內容，不修改 CLAUDE.md 或 hook
- **模組化優先**：每個功能以獨立模組實現，透過 `opentree.json` 宣告依賴和介面
- **靜態 + 動態分離**：模組 `rules/*.md` 合併為靜態 CLAUDE.md；動態資訊（日期、身份、頻道）由 `--system-prompt` 注入

## 輸出品質自檢清單

> **高優先級規則 -- 每次最終輸出前必須逐項確認**

1. **數量達標**：若使用者要求「至少 N 則/個/項」，數量必須 ≥ N
   - 數量不足時，**必須繼續搜尋/蒐集**，不可直接結束
   - 「至少 N 則」是 hard requirement，不是參考值
   - 多輪蒐集策略：第一輪找一批 → 第二輪補缺口 → 第三輪去重整理
2. **來源可查**：重要判斷有可驗證來源（URL 或具體出處）
3. **數據支撐**：關鍵分析結論有具體數字或量化依據，不只是概念性描述
4. **實際交付**：產生的檔案已透過 upload-tool 上傳 Slack，不能只回覆本機路徑
5. **格式正確**：Slack 回覆用 Slack 格式；Markdown/HTML 文件用對應格式（見 message-format.md）
