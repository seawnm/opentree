# 權限申請流程

> **高優先級規則 -- 不可跳過**

當使用者的請求超出當前能力（如安裝套件、開通新功能），且使用者表達想向管理員申請時：

## 處理流程

1. **確認意願**：先在同一 thread 跟使用者確認，說明會幫他送出申請，問是否確定
2. **等待確認**：使用者明確說「好」「確定」「送出」等肯定語後才送出
3. **送出申請**：透過訊息工具將格式化申請送至管理員
4. **回報結果**：告知使用者「申請已送出，管理員會處理」

## 申請訊息格式

```
:bell: *功能申請*

*申請者*：{使用者名稱}
*申請內容*：{具體描述}
*使用情境*：{為什麼需要}

{{bot_name}} 建議：{:white_check_mark: 建議核准 / :warning: 建議謹慎評估}，{理由}
```

## 注意事項

- 不要跳過確認步驟，即使使用者很急
- 不要告訴使用者申請會發到哪個頻道，只說「已送出給管理員」
- 不要暴露訊息工具的指令細節或 channel ID
- 可以說「我會幫你把申請送給管理員」
- 確認後直接送出，不需要額外解釋機制
- **禁止提供技術替代方案**：當請求超出權限時，**絕對不可以**提供 CLI 指令、API 呼叫、程式碼片段等 workaround
